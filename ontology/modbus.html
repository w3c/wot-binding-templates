Object Properties

Byte Encoding

IRI: https://www.w3.org/2019/wot/modbus#encodingOption

Little Endian or Big Endian encoding of bytes, i.e. the bit order within each register. Default is Big Endian

Entity

IRI: https://www.w3.org/2019/wot/modbus#entity

A registry type to let the runtime automatically detect the right function code

First DoubleWord Low Flag

IRI: https://www.w3.org/2019/wot/modbus#firstDWordLow

Data encoding for 64-bit values; same as firstWordLow but based on DWORDS (i.e. 2 registers) instead of WORDS (1 register)

First Word Low Flag

IRI: https://www.w3.org/2019/wot/modbus#firstWordLow

Data encoding for 32-bit values to chose which of the register is the low value

Function

IRI: https://www.w3.org/2019/wot/modbus#function

Function Code sent by the client in every request. Specifying the desired interaction.

Length of requested registers/coils

IRI: https://www.w3.org/2019/wot/modbus#length

Specifies the amount of either registers or coils to be read/written

Offset

IRI: https://www.w3.org/2019/wot/modbus#offset

Specifies the starting address of the modbus operations

Request Options

IRI: https://www.w3.org/2019/wot/modbus#options

Request Options

Modbus TCP maximum polling rate

IRI: https://www.w3.org/2019/wot/modbus#pollingTime

The Modbus specification does not define a maximum or minimum allowed polling rate, however specific implementations might introduce such limits. Defined as integer of milliseconds.

Response Code

IRI: https://www.w3.org/2019/wot/modbus#responseCode

Response Code returned by the server after a request

Modbus response maximum waiting time

IRI: https://www.w3.org/2019/wot/modbus#timeout

Defines how much time the runtime should wait until it receives a reply from the device.

UnitID

IRI: https://www.w3.org/2019/wot/modbus#unitID

The Unit ID is usually not needed for ModbusTCP, since the IP-address works as unique identifier, but for compability reasons still often included

Write Value

IRI: https://www.w3.org/2019/wot/modbus#writeValue

The value to write to the register or coil, encoded as integer, omitted and ignored for reading operations

Zero Based Addressing Flag

IRI: https://www.w3.org/2019/wot/modbus#zeroBasedAddressing

Modbus implementations can differ in the way addressing works, as the first coil/register can be either referred to as '0' or '1'. Default is '0'.
